# Code Style

It is important that your code is clean, well-named, well organized, commented, great DX, and modular. A master dev with maximum scrutiny & attention to detail should highly approve. 

## Comments

Add comments, especially where behavior or rationale behind decisions might be unclear, or to tie code back to specific points in the spec.

## Specialization

- If you're operating on a TypeScript/JavaScript/Node/Web project, read [javascript.md](javascript.md)
- If you're working with Electron, read [electron.md](electron.md)
